# Glossary

## AEAD

Authenticated encryption with associated data. A symmetric encryption mode that
provides confidentiality and integrity.

## Algorithm Agility

The ability to change cryptographic algorithms without redesigning the whole
application or protocol.

## Backend

A concrete cryptographic implementation behind the PQC Bridge API.

## C ABI

A stable C application binary interface used as the boundary for language
bindings.

## FIPS 140-3

A validation program for cryptographic modules. It is different from algorithm
standardization.

## FIPS 203

NIST standard for ML-KEM.

## FIPS 204

NIST standard for ML-DSA.

## FIPS 205

NIST standard for SLH-DSA.

## HKDF

HMAC-based key derivation function. It derives cryptographic keys from shared
secret material.

## Hybrid Key Agreement

A key agreement construction that combines classical and post-quantum
cryptography.

## KAT

Known-answer test. A deterministic test vector used to verify implementation
behavior.

## KEM

Key encapsulation mechanism. A public-key primitive used to establish a shared
secret.

## ML-DSA

Module-Lattice-Based Digital Signature Algorithm, standardized by NIST FIPS 204.

## ML-KEM

Module-Lattice-Based Key Encapsulation Mechanism, standardized by NIST FIPS 203.

## PQC

Post-quantum cryptography. Cryptography designed to resist attacks from both
classical and quantum computers.

## SLH-DSA

Stateless Hash-Based Digital Signature Algorithm, standardized by NIST FIPS 205.

## X25519

A classical elliptic-curve Diffie-Hellman key agreement mechanism widely used in
modern protocols.
